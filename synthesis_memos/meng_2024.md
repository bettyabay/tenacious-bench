# Memo: Hong et al., 2024 — ORPO: Monolithic Preference Optimization without Reference Model

**Paper:** "ORPO: Monolithic Preference Optimization without Reference Model" — Hong et al., 2024  
**Relevance:** Selected training method for Path B judge (adopted over DPO after mentor review)  
**Length:** ~1 page

---

## Key Claims

ORPO introduces an odds-ratio penalty term that is appended directly to the standard
SFT cross-entropy loss, eliminating the need for a separate reference model:

```
L_ORPO = L_SFT + λ · L_OR

where L_OR = -log σ( log (p_θ(y_w|x) / (1 - p_θ(y_w|x)))
                   - log (p_θ(y_l|x) / (1 - p_θ(y_l|x))) )
```

The authors claim (§3.2):
1. ORPO matches or outperforms DPO on Alpaca-Eval and MT-Bench without a reference model
2. The monolithic loss is more memory-efficient (one forward pass, one model copy)
3. Training is more stable on small datasets because the SFT component prevents
   the policy from collapsing to degenerate outputs when chosen/rejected outputs are similar
4. The λ hyperparameter is simpler to tune than DPO's β because it has an intuitive
   interpretation (relative weight of preference vs language-modelling loss)

---

## Comparison to DPO (Rafailov 2023)

| Aspect | DPO | ORPO |
|--------|-----|------|
| Reference model | Required (doubles memory) | Not required |
| Loss components | Single preference term | SFT + odds-ratio (joint) |
| Hyperparameter | β (KL penalty weight) | λ (odds-ratio weight) |
| Small-dataset stability | Can collapse if pairs too similar | SFT term stabilises training |
| Implementation (TRL) | `DPOTrainer` | `ORPOTrainer` |
| VRAM on T4 (8B model) | ~14–15 GB | ~10–12 GB |

## Design Choice I Disagree With (§4.1 — Benchmark Evaluation Protocol)

The paper evaluates ORPO exclusively on general-purpose benchmarks (Alpaca-Eval 2.0,
MT-Bench, IFEval) using model responses in the 1,000–10,000 pair range. The authors
conclude (§4.1): *"ORPO consistently outperforms DPO across all evaluated benchmarks"*
and present this as evidence of broad applicability.

**I disagree with this generalisation for domain-specific, rule-based preference tasks.**

The benchmarks used in the paper measure open-ended instruction-following quality, where
the preference signal is based on human rater preference between two plausible responses.
In my setting, the preference signal is *deterministic* — a suppression rule either fires
or it doesn't. A pair like (chosen=suppress, rejected=send) where the only difference is
the presence of `disqualifiers: ["anti_offshore"]` in the context is not a question of
stylistic preference; it is a binary correctness decision derivable mechanically from the
rule set.

My Week 10 evidence supports a stronger claim: the three tasks that failed all five τ²-Bench
trials (tasks 76, 92, 104 — simulations `0857ba6e`, `19d13ac9`, `0c380837` and their
replicas) are not cases where the agent produced a *worse* output — they are cases where
the agent produced the *wrong class of output entirely*. The chosen/rejected distinction is
not a matter of degree but of category. For this type of preference signal, the SFT component
of ORPO's joint loss is actually a risk: training the model to *produce* the chosen completion
fluently could reinforce generation even in block cases, rather than training it to abstain.

In practice, I mitigate this by formatting the chosen completion as a structured JSON
verdict `{"decision": "BLOCK", "score": 0.0}` rather than as a natural-language email —
this prevents the SFT loss from reinforcing fluent outreach text. But the paper's evaluation
protocol would not have surfaced this concern because its preference pairs are all
open-ended text generation tasks, not rule-application tasks.

**The implication for my project:** ORPO's λ tuning and no-reference-model simplicity
are genuine advantages for the T4 VRAM constraint and small dataset size. However, I should
monitor the SFT loss component separately during training — if it decreases faster than the
odds-ratio component, the model may be learning to generate fluent verdicts of any class
rather than learning the correct class boundary.

---

## Caveats

- The paper does not study preference datasets under 500 pairs; the 323-pair dataset in
  this project is extrapolating beyond the paper's empirical range
- λ=0.1 (default from the paper) is tested only on general-purpose models; may need
  adjustment for a judge-role fine-tune on a domain-specific task
- ORPO has no equivalent to DPO's reference model divergence metric; the training
  monitor must rely on the SFT and odds-ratio loss components separately to detect collapse

---

## Decision

**ORPO adopted.** Primary reasons: T4 VRAM constraint eliminates reference model option;
323-pair dataset is small enough that the joint SFT loss provides useful regularisation;
λ is easier to sweep than β for a binary task. The SFT-component risk is mitigated by
formatting the chosen completion as a structured JSON verdict, not as natural-language text.
